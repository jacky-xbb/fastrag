# 本地问答知识库 — 调研文档

> 目标：做一个本地运行的问答知识库
> 1. 可以导入 PDF
> 2. 可以对话，检索出对应的问答
> 3. 保留会话历史记录
> 4. 可以上网（联网兜底）

---

## 1. 技术栈结论

| 层 | 选型 | 说明 |
|----|------|------|
| 框架 | **Mastra**（TypeScript） | 一站式覆盖 RAG / Agent / Memory / Tools |
| 向量库 | **libSQL**（本地文件，Mastra 默认） | 一个文件搞定向量 + 会话历史，零基础设施 |
| PDF 解析 | `unpdf` / `pdftotext`（有文字层） | Mastra 不解析 PDF，需自己做 |
| **OCR** | **PaddleOCR-VL-1.6**（托管 API，直接吃 PDF） | **必须**，因 94% 是扫描件；实测三家最优，见 ADR-0003 |
| Embedding | OpenRouter `openai/text-embedding-3-small`（1536维） | Mastra 内建调用，无需单独库 |
| 对话 LLM | OpenRouter `openai/gpt-5` | 与 embedding 同一个 OpenRouter key |
| 会话历史 | Mastra `Memory` + libSQL | 与向量库**同一个文件** |
| 联网 | Tavily web search | Agent 按需调用 |

---

## 2. 数据现状（已实测）

- 目录：`pdf/`，共 **18 个 PDF / 约 460 页**（防水卷材类国标）
- 实测抽取文字结果：

| 类型 | 数量 | 说明 |
|------|------|------|
| ✅ 有文字层（可直接抽） | **1** | GBT 18242-2025 |
| ❌ 扫描件（必须 OCR） | **17 / ~445 页** | 其余全部，含 293 页《JTST 232-2019 水运工程材料试验规程》 |

> 检测方法：`pdfplumber` 采样首页 + 中间页字符数，扫描件均为 0 字符，已交叉验证非「封面无字」误判。

**结论：不做 OCR，知识库 94% 内容是空的。OCR 是项目关键路径。**

---

## 3. 关键决策（已确认）

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 向量库 | **libSQL**（非 MongoDB） | 自己小范围用、数据量小（约 2000~5000 向量）、要存历史 → libSQL 最省心，比 MongoDB 少一个 Docker 容器 |
| 模型来源 | **云端 API** | 中文效果好、配置简单；项目本身允许联网 |
| OCR | **PaddleOCR-VL-1.6**（托管 API） | 实测 vs Mistral OCR / PP-OCRv5：文字最准、指标表格输出带合并单元格的 HTML 最忠实（见 ADR-0003）；直接吃 PDF，免本地渲染 |
| 扫描件处理 | 先检测再 OCR | 已检测完成（见上） |

---

## 4. libSQL vs pgvector（为什么选 libSQL）

| 维度 | libSQL | pgvector |
|------|--------|----------|
| 本质 | SQLite 分支，向量内建 | Postgres 扩展 |
| 部署 | **一个文件**，零服务 | 要跑 Postgres 服务 |
| 成熟度 | 较新 | 非常成熟 |
| 并发写 | 弱（单写者） | 强 |
| 复杂过滤/规模 | 一般 | 强、后劲足 |
| 会话历史 | **同文件搞定** | 同库也可 |
| Mastra 集成 | **默认** | 一等支持 |

**判断**：本项目「自己小范围用 + 小数据量 + 单用户」，libSQL 的短板（并发弱、超大规模吃力）都用不到，故选 libSQL。需扩多人/高并发/复杂过滤时再换 pgvector。

---

## 5. 本地 libSQL → 生产 Turso，切换成本：**极低**

Mastra 的 `LibSQLVector` 本地与 Turso **同一套 API**（`createIndex / upsert / query / describeIndex / deleteIndex` 完全一致）。切换只改构造参数，**业务代码不动**：

```ts
// 本地
new LibSQLVector({ connectionUrl: "file:./vector.db" })

// 生产 Turso：仅加 authToken，可选 syncUrl/syncInterval 做复制
new LibSQLVector({
  connectionUrl: "libsql://your-db.turso.io",
  authToken: process.env.TURSO_AUTH_TOKEN,
})
```

唯一额外动作：把数据搬过去（重跑一次入库脚本，或 dump/restore）。→ **本地起步无后顾之忧。**

---

## 6. 是否需要 OCR / embedding 库

| 项 | 结论 | 原因 |
|----|------|------|
| **OCR** | ✅ 必须，需外部方案 | 17/18 为扫描件；Mastra 不解析 PDF、不做 OCR。采用 PaddleOCR-VL 托管 API（见 ADR-0003） |
| **PDF 解析** | ✅ 需外部 | 有文字层的用 `unpdf`/`pdftotext`；Mastra 只接收 text/markdown |
| **embedding 库** | ❌ 不用单独装 | Mastra 内建，经 Vercel AI SDK `embedMany()` + 模型串 `'openai/text-embedding-3-small'` 调用；走 OpenRouter provider，与对话共用一个 key |
| **切块 / 检索 / 记忆** | ❌ 不用单独装 | Mastra 全包（`MDocument` + 9 种切块策略 + `createVectorQueryTool` + `Memory`） |

> **外部只需补「PDF 解析 + OCR」这一段，其余 Mastra 全覆盖。**

---

## 7. Mastra 切块 / Embedding 用法要点

- **文档抽象**：`MDocument.fromText() / fromMarkdown() / fromHTML() / fromJSON()`（**无 fromPDF**）
- **切块策略**（9 种）：`recursive`（结构化，默认推荐）、`markdown` / `semantic-markdown`、`token`、`sentence`、`character`、`html` / `json` / `latex`
- **常用参数**：`maxSize`、`overlap`
- **Embedding**：`embedMany()` + `'openai/text-embedding-3-small'`；text-embedding-3 支持自定义维度以省存储

## 8. 整体数据流

```
                  ┌─ 有文字层(1个) ── unpdf/pdftotext 直接抽 ──┐
pdf/ (460页) ─────┤                                          ├─→ markdown(带页码来源)
                  └─ 扫描件(17个) ── PaddleOCR-VL API(直接吃PDF) ─┘     │
                                                                      │
                                              MDocument + recursive 切块
                                                                      │
                                              embedMany (text-embedding-3-small)
                                                                      │
                                                   libSQL 向量库 (file:./vector.db)
                                                                      │
用户提问 ── Agent ─┬─ vectorQueryTool ── 检索国标知识库 ───────────────┘
                   ├─ webSearchTool ──── 联网兜底
                   └─ Memory ─────────── 多轮历史 (同一个 libSQL 文件)
```

## 9. 落地步骤（含验证点）

1. **环境**：`npm create mastra` 初始化，配置 libSQL（本地文件）+ 云端 API key
   → 验证：能建向量索引、能连库
2. **PDF → 文本**：有文字层直接抽；扫描件 → PaddleOCR-VL API → markdown（表格为 HTML，保留**文件名 + 页码**作来源）
   → 验证：抽查几页 OCR 准确率（尤其表格/指标）
3. **入库**：`MDocument` 切块 → `embedMany` → `upsert` 到 libSQL
   → 验证：向量检索能返回相关片段
4. **Agent**：挂 `createVectorQueryTool` + webSearch + `Memory`
   → 验证：问某个国标指标能答对且带来源、多轮能记住上下文
5. **界面**：Mastra dev playground 或简单 web UI

## 10. 待确认 / 风险

- **API key（已定，3 个）**：LLM（OpenRouter，对话 gpt-5 + embedding 3-small 共用）、OCR（PaddleOCR-VL）、联网（Tavily）。embedding 已收口 OpenRouter，不再单连 OpenAI（见 ADR-0001）。
- **OCR 成本**：约 445 页一次性 PaddleOCR-VL，离线入库、建好库不再跑；按页计费，单价以 aistudio-app 账单为准。
- **OCR 质量**：国标表格/公式密集，需人工抽查；必要时对关键文件人工校对。
- **来源可追溯**：入库时务必保留「文件名 + 页码」元数据，便于答案标注出处。

## 11. OpenRouter Provider 接入（对话 + embedding 共用一个 key）

> 关键：Mastra/AI SDK 里默认写 `'openai/text-embedding-3-small'` 会**直连官方 OpenAI**。要走 OpenRouter，必须显式建 OpenRouter provider 再把它的模型实例传进去——不是改个 key 就行。OCR（PaddleOCR-VL）和联网（Tavily）走各自的 API，不经过这里。

安装：

```bash
npm i @openrouter/ai-sdk-provider
```

建 provider（读 `OPENROUTER_API_KEY`），导出对话 + embedding 两个模型实例：

```ts
// src/lib/openrouter.ts
import { createOpenRouter } from '@openrouter/ai-sdk-provider'

export const openrouter = createOpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY!,
})

export const chatModel  = openrouter.chat('openai/gpt-5')                          // 对话
export const embedModel = openrouter.textEmbeddingModel('openai/text-embedding-3-small') // embedding（1536 维）
```

三处用到它：

```ts
// 1) Agent 对话
import { Agent } from '@mastra/core/agent'
const agent = new Agent({ name: '国标问答', model: chatModel /* tools, memory */ })

// 2) 入库时批量算向量
import { embedMany } from 'ai'
const { embeddings } = await embedMany({ model: embedModel, values: chunks.map(c => c.text) })

// 3) 检索工具（查询侧也得用同一个 embedding 模型，否则向量空间对不上）
import { createVectorQueryTool } from '@mastra/rag'
const vectorQueryTool = createVectorQueryTool({
  vectorStoreName: 'libsql',
  indexName: 'standards',
  model: embedModel,
})
```

落地时注意：① 入库和检索**必须用同一个 embedding 模型**（维度/空间一致）；② 包是 AI SDK v6 接口，`npm create mastra` 后确认 Mastra 的 AI SDK 大版本对得上；③ 模型串 `openai/gpt-5`、`openai/text-embedding-3-small` 以 OpenRouter 当时的可用型号为准，开建前在 OpenRouter 模型页核一下。
